《Substrate Docs》翻译 1.1 Getting Started - Welcome

Welcome（欢迎）

这是 Substrate 开发人员的官方文档中心。 此资源的目的是帮助读者了解使用 Substrate 进行区块链开发的复杂性。 本指南分为几个部分，解释了构建 Substrate 的原则和设计策略，以及成为高效的 Substrate 区块链开发人员所需的特定技能。

信息

为了充分利用 Substrate，您应该对计算机科学和基本的区块链概念有很好的了解。 头、块、客户端、哈希、交易和签名等术语应该很熟悉。 Substrate 建立在 Rust 编程语言之上，它利用新颖的设计模式来实现安全快速的代码开发。 尽管您无需了解 Rust 即可开始使用 Substrate，但对 Rust 的良好理解将使您成为更好的 Substrate 开发人员。 查看 [Rust 社区](Learn Rust - Rust Programming Language (rust-lang.org))提供的优秀资源，以培养您的 Rust 开发技能。

Substrate 采用模块化方法进行区块链开发，并定义了一组丰富的原语，允许开发人员使用强大的、熟悉的编程习惯用法。

使用

Substrate 被设计于三种情况下使用：

​ 1.使用 Substrate 节点：您可以运行预先设计的 Substrate 节点并配置其创世块。在这种情况下，您只需要提供一个 JSON 文件并启动您自己的区块链。 JSON 文件允许您配置组成 Substrate 节点运行时的模块的创世状态，例如：Balances、Staking 和 Sudo。您可以在Create Your First substrate Chain（创建您的第一条Substrate链）和Start a Private Network（创建一个私有网络）教程中了解有关运行 Substrate 节点的更多信息。

​ 2.使用 Substrate FRAME：您可以使用 FRAME（模块化实体的运行时聚合框架）轻松创建自己的自定义运行时，该框架用于构建 Substrate 节点。这为您提供了对区块链逻辑的大量自由，并允许您配置数据类型、从模块库（称为“Pallet”）中进行选择，甚至添加您自己的自定义Pallet。 Substrate Developer Hub 节点模板是此类项目的有用起点。要了解更多信息，请参阅Build a dApp （创建一个去中心化App）和Add a Pallet（添加一个Pallet）的教程。

​ 3.使用 Substrate Core：可以忽略整个 FRAME 系统，可以从头开始设计和实现运行时。这可以用任何可以针对 WebAssembly 的语言来完成。如果运行时可以与 Substrate 节点的抽象块创作逻辑兼容，那么您可以简单地从您的 Wasm blob 构建一个新的创世块，并使用现有的基于 Rust 的 Substrate 客户端启动您的链。如果没有，那么您将需要更改客户端的块创作逻辑，甚至可能更改标头和块序列化格式。就开发工作而言，这是迄今为止使用 Substrate 最困难的方式，但也为您提供了最大的创新自由。
