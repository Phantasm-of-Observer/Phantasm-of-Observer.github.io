Installation（安装）

此页面将指导您完成为 Substrate 开发准备计算机所需的 2 个步骤。 由于 Substrate 是使用 Rust 编程语言构建的，您需要做的第一件事就是为 Rust 开发准备计算机——这些步骤将根据计算机的操作系统而有所不同。 配置 Rust 后，您将使用其工具链与 Rust 项目进行交互； Rust 工具链的命令对于所有受支持的基于 Unix 的操作系统都是相同的。

WebAssembly 编译

Substrate 使用 WebAssembly (Wasm) 来生成可移植的区块链运行时。 您需要将 Rust 编译器配置为使用 [nightly builds](G - How Rust is Made and “Nightly Rust” - The Rust Programming Language (rust-lang.org))（nightly构建），以允许您将 Substrate 运行时代码编译到 Wasm 目标。

消息

在所有 Substrate 可以使用稳定的 Rust 工具链之前，需要解决 Rust 的上游问题。 如果您对解决此问题的原因和方式感到好奇，[这就是我们的跟踪问题](Compile the wasm runtime with a stable toolchain · Issue #1252 · paritytech/substrate (github.com))。

Substrate master 的最新nightly版本

正在构建 Substrate 本身的开发人员应该始终使用 Rust stable 和 nightly 的最新无错误版本。 这是因为 Substrate 代码库遵循 Rust nightly 的提示，这意味着 Substrate 的更改通常取决于 Rust nightly 编译器的上游更改。 为了确保您的 Rust 编译器始终是最新的，您应该运行：

摘要

如果上游 Substrate 代码库中的更改取决于 Rust 编译器的新功能，则可能需要偶尔重新运行 rustup 更新。 当你这样做时，你的 nightly 和 stable 工具链都将被拉到最新的版本，并且对于 nightly，通常不会编译 WASM 没有错误（尽管它经常这样做）。 如果您从 rustup 中获得 WASM 构建错误并根据需要nightly降级，请务必指定您的nightly版本。

Rust nightly工具链

如果你想保证你的构建在你更新 Rust 和其他依赖项时在你的计算机上运行，你应该使用一个特定的 Rust nightly 版本，该版本已知与他们正在使用的 Substrate 版本兼容； 此版本因项目而异，不同的项目可能会使用不同的机制将此版本传达给开发人员。 例如，Polkadot 客户端在其[发行说明](Releases · paritytech/polkadot (github.com))中指定了此信息。
