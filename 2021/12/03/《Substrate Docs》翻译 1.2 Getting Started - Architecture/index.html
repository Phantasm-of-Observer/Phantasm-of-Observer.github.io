Architecture（结构）

Substrate 客户端是一个运行基于 Substrate 的区块链节点的应用程序。 它由多个组件组成，包括但不限于以下内容：

  • 存储：用于保持 Substrate 区块链的演化状态。区块链网络允许参与者就存储状态达成无需信任的 consensus（共识）。 Substrate 附带一个简单而高效的 key-value storage mechanim（键值对存储机制）。
  • 运行时：定义如何处理块的逻辑，包括状态转换逻辑。在 Substrate 中，运行时代码被编译为 Wasm 并成为区块链存储状态的一部分。这实现了基于 Substrate 的区块链的定义特征之一：forkless runtime upgrades（无分叉运行时升级）。 Substrate 客户端还可能包含一个“原生运行时”，它是为与客户端本身（而不是 Wasm）相同的平台编译的。将调用分派到运行时的客户端组件称为 executor（执行器），其作用是在本机代码和解释型 Wasm 之间进行选择。尽管本机运行时可能提供性能优势，但如果它实现了较新的 version（版本），执行程序将选择解释 Wasm 运行时。
  • P2P网络：允许客户端与其他网络参与者进行通信的能力。 Substrate 使用 libp2p 网络堆栈的 Rust 实现来实现这一点。
  • 共识：允许网络参与者就区块链状态达成一致的逻辑。 Substrate 使得提供自定义共识引擎成为可能，并且还附带了几种建立在 [Web3 基金会](Research at Web3 Foundation — Research at W3F)研究之上的共识机制。
  • RPC（远程过程调用）：允许区块链用户与网络交互的能力。 Substrate 提供 HTTP 和 WebSocket RPC 服务器。
  • Telemetry：嵌入式 [Prometheus](Prometheus - Monitoring system & time series database)服务器公开的客户端指标。
