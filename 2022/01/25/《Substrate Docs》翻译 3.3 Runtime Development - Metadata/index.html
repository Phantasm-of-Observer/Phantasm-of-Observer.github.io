元数据（Metadata）

您在 Substrate 上构建的区块链会公开元数据，以便更轻松地与它们交互。 此元数据由形成您的区块链的不同[托盘](FRAME | Substrate_)分隔。

对于每个托盘，元数据提供有关该托盘公开的[存储项](Storage | Substrate_)、[外部调用](Extrinsics | Substrate_)、[事件](Events | Substrate_)、常量和错误的信息。 Substrate 会自动为您生成此元数据，并通过 RPC 调用使其可用。

由于 Substrate 区块链的运行时是区块链状态的一个不断发展的部分，因此区块链元数据存储在每个块的基础上。 请注意，查询旧块的元数据（例如，使用存档节点）可能会导致获取与区块链当前状态不兼容的过时元数据。 如[升级](Upgrades | Substrate_)文档中所述，在 Substrate 链之上构建的开发人员可以预期，链的元数据仅应在链的运行时 spec_versionRuntimeVersion in sp_version - Rust (substrate.io)更改时更改。

本文档中的所有示例均取自 Kusama 上的 1,768,321 区块。 您可以在阅读本文档的其余部分之前查看完整的元数据，并在继续进行时继续参考它。

获取元数据（Get the metadata）

有许多特定于语言的库可用于从 Substrate 节点获取元数据，以及与语言无关的 HTTP 和 WebSocket API。

Rust

获取元数据的最简单方法是查询自动生成的 JSON-RPC 函数 state_getMetadata。 这将返回一个 SCALE 编码字节的向量。 您可以使用 frame-metadata 和[parity-scale-codec](parity_scale_codec - Rust (substrate.io)) 库对此进行解码。

一些有用的库，如[substrate-subxt](paritytech/subxt: Submit extrinsics (transactions) to a substrate node via RPC (github.com))获取元数据并为您解码。 解码后，可以使用[serde](Overview · Serde)将结构序列化为 JSON。 如果您希望更直接地使用 RPC，可以使用[JSONRPC](paritytech/jsonrpc: Rust JSON-RPC implementation (github.com))和[jsonrpsee](paritytech/jsonrpsee: Rust JSON-RPC library on top of async/await (github.com)) Rust 库。

JavaScript

如果你使用 JavaScript，[polkadot-js/api](Overview | polkadot{.js}) 提供了与 Substrate 区块链交互的 API，包括 [getMetadata](JSON-RPC | polkadot{.js}) 函数。

元数据格式（Metadata formats）

本节将简要回顾以十六进制字符串表示的 SCALE 编码元数据，然后更详细地了解元数据的解码格式。

编码元数据格式（Encoded metadata format）

JSON-RPC state_getMetadata 方法返回的十六进制 blob 以硬编码的幻数 0x6d657461 开头，它表示纯文本中的“元”。 下一条数据（上例中的 0x0b）代表元数据版本； 解码十六进制值 0x0b 产生十进制值 11，这是结果编码的 Substrate 元数据格式的版本。 在元数据版本之后，结果字段中编码的下一条信息是通知区块链运行时的托盘数量； 在上面的示例中，十六进制值 0x7c 表示十进制数 31，通过采用其二进制表示（十六进制表示为 11111 或 0x1F），将其向左移动两位 (1111100) 并将其编码为十六进制，对其进行 SCALE 编码。

剩下的 blob 对[每个托盘的元数据](Page not found · GitHub Pages (substrate.io))进行编码，下面将对其进行审查，以及一些[外部元数据](Page not found · GitHub Pages (substrate.io))，这些元数据大部分超出了本文档的范围。
