《Substrate Docs》翻译 2.2 Key Concepts - Extrinsics

Created2022-01-09|Updated2022-03-04|《Substrate Docs》翻译
|Post View:

外在的（Extrinsics）

extrinsic是来自链外并包含在块中的一条信息。Extrinsics分为三类：固有交易、签名交易和未签名交易。

请注意，[事件](Events | Substrate_)不是外在的。 链为链本身固有的信息片段发出事件。 例如，质押奖励是事件，而不是外在的，因为奖励是由链逻辑的内在环境触发的。

区块结构（Block structure）

Substrate 中的区块由区块头和外部数组组成。区块头包含块高度、父哈希、外部根、状态根和摘要。 本节将只关注外部根。

外在元素被捆绑到一个区块中，作为一系列在运行时中定义的要执行的区块。 外部根是本系列的密码摘要。 这有两个目的。 首先，它可以防止在构建和分发区块头之后对一系列外部参数进行任何更改。 其次，它提供了一种方法，允许轻客户端简洁地验证任何给定的外部数据确实存在于一个只知道区块头的区块中。

  • [Block Reference](Block in sp_runtime::traits - Rust (substrate.io))

内在的（Inherents）

Inherents是未签名的信息片段，仅由区块作者插入到块中。 它们不会在网络上传播或存储在交易队列中。 从技术上讲，没有什么可以阻止传播inherents的 Substrate 链，但不会有基于费用的垃圾邮件预防机制。

Inherents表示以一种自以为是的方式描述许多有效信息之一的数据。 仅仅因为足够多的验证者已经同意它们是合理的，它们就被认为是“真实的”。

例如，区块的作者可能会在区块中插入一个inherent时间戳。没有办法证明时间戳是真实的，就像通过签名证明发送资金的目的一样。 相反，验证者根据其他验证者对该时间戳的合理程度来接受或拒绝该块，这可能意味着它在他们自己的系统时钟的某个可接受范围内。

  • [Inherents Reference](sp_inherents - Rust (substrate.io))

已签名交易（Signed transactions）

签名交易包含发出交易的账户的签名，并且需要支付费用才能将交易包含在链上。 因为可以在执行之前识别在链上包含签名交易的价值，所以它们可以在节点之间的网络上传播，垃圾邮件的风险很低。

签名交易符合以太坊或比特币交易的概念。

未签名的交易（Unsigned transactions）

有些情况需要未签名的交易。 谨慎使用未签名的交易，因为它们的验证逻辑可能很困难。

由于交易未签名，因此没有人支付费用。正因为如此，交易队列缺乏防止垃圾邮件的经济逻辑。未签名的交易也缺乏随机数，使得重放保护变得困难。 一些交易需要使用未签名的变体，但它们需要某种形式的基于[签名扩展](Transaction Pool | Substrate_)的自定义实现的垃圾邮件预防，这可以存在于未签名的交易中。

Substrate 中未签名交易的一个示例是权威发送的[I’m Online](FRAME | Substrate_)心跳交易（heartbeat transaction）。该交易包括来自会话密钥的签名，该密钥不控制资金，因此无法支付费用。 交易池通过检查是否已经在会话中提交了心跳来控制垃圾邮件。
