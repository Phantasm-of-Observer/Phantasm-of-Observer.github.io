《Substrate Docs》翻译 2.5 Key Concepts - Session Keys

会话密钥（Session Keys）

Substrate提供了[Session pallet](pallet_session - Rust (substrate.io))，它允许验证者管理他们的会话密钥。

会话密钥是验证者用来签署共识相关消息的“热密钥”。 它们不应用作控制资金的帐户密钥，而应仅用于其预期目的。 它们可以定期更换； 您的控制器只需要通过签署会话的公钥并通过外部广播此证书来创建新的会话密钥。 会话键也被通用定义并在运行时具体化。

要创建会话密钥，验证者操作员必须证明密钥代表他们的 Stash 帐户（权益）和提名人行事。 为此，他们通过使用控制器密钥对密钥进行签名来创建新的会话密钥。 然后，他们通过在链上的交易中发布会话证书来通知链该密钥代表他们的控制器密钥。

实现（Implementation）

验证者使用会话密钥来签署与共识相关的消息。[SessionKeys](SessionKeys in sp_session - Rust (substrate.io))是一种通用的、可索引的类型，在运行时具体化。

您可以声明任意数量的会话密钥。 例如，默认的 Substrate 节点使用四个。 其他链可能有更多或更少，具体取决于链期望其验证者执行的操作。

在实践中，验证者将所有会话公钥合并到一个对象中，使用“控制器”帐户签署公钥集，并提交交易以在链上注册密钥。 这种链上注册将验证节点与持有资金的账户联系起来。 因此，该帐户可以根据节点的行为获得奖励或削减。

强类型包装器（Strongly typed wrappers）

来自 Substrate 节点的会话密钥可以使用相同的加密技术，但在运行时逻辑中用于非常不同的目的。 为了防止错误的键被用于错误的操作，强 Rust 类型包装了这些键，使它们彼此不兼容，并确保它们仅用于其预期目的。

生成和使用（生成和使用）

作为节点操作员，您可以使用 RPC 调用[author_rotateKeys](AuthorApi in sc_rpc::author - Rust (substrate.io))生成密钥。 然后，您需要使用[session.setKeys](Module in pallet_session - Rust (substrate.io))事务在链上注册新密钥。

重要信息（Important）

为了提高安全性，每次会话都应更改会话密钥。 这可以通过创建新的会话密钥、将会话公钥放入外部密钥并将此外部密钥应用到链上来完成。

由于会话密钥是必须在线保持的热键，因此不应使用单个密钥来控制资金。 处理会话密钥的所有逻辑都在Substrate客户端、原语和Session pallet中。 如果其中一个Session密钥被泄露，攻击者可能会做出slashable行为。
