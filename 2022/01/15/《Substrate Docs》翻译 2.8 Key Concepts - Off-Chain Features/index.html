《Substrate Docs》翻译 2.8 Key Concepts - Off-Chain Features

Created2022-01-15|Updated2022-03-04|《Substrate Docs》翻译
|Post View:

链下功能（Off-Chain Features）

通常需要查询和/或处理链下数据，然后才能将其包含在链上状态中。 执行此操作的传统方法是通过[预言机](Glossary | Substrate_)。 虽然这种方法有效，但在安全性、可扩展性和基础架构效率方面仍然存在一些缺陷。

为了使链下数据集成更安全、更高效，Substrate 提供了链下功能：

  • 链下工作机（Off-Chain Worker，即OCW） 子系统允许执行长时间运行且可能不确定的任务（例如 Web 请求、数据加密/解密和签名、随机数生成、CPU 密集型计算、链上数据的枚举/聚合 等），否则可能需要比块执行时间更长的时间。
  • 链下存储（Off-Chain Storage）提供 Substrate 节点的本地存储，可以由链下工作人员（读取和写入访问）和链上逻辑（通过链下索引进行写入访问，但不能读取访问）访问。 这允许不同的工作线程相互通信并存储不需要在整个网络上达成共识的用户或节点特定数据。
  • 链下索引（Off-Chain Indexing）允许运行时（如果选择加入）独立于 OCW 直接写入链下存储。 这用作链上逻辑的本地/临时存储，并补充其链上状态。

链下功能在 Substrate 运行时之外的自己的 Wasm 执行环境中运行。 这种关注点分离确保了区块生产不受长期运行的链下任务的影响。 然而，由于链下特性在与运行时相同的代码中声明，它们可以轻松访问链上状态以进行计算。

链下工作机（Off-chain workers）

链下工作机可以访问扩展 API 以与外部世界进行通信：

  • 能够向链提交交易（签名或未签名）以发布计算结果。
  • 一个功能齐全的 HTTP 客户端，允许工作人员从外部服务访问和获取数据。
  • 访问本地密钥库以签署和验证语句或交易。
  • 在所有链下工作人员之间共享的附加本地键值数据库。
  • 用于随机数生成的安全本地熵源。
  • 访问节点的精确本地时间。
  • 睡眠和恢复工作的能力。

请注意，来自链下工作人员的结果不受常规交易验证的约束。 应实施验证机制（例如投票、平均、检查发送者签名或简单地“信任”）以确定哪些信息进入链中。

链下存储（Off-chain storage）

顾名思义，存储不存储在链上。 它可以被链下工作线程（读写访问）和链上逻辑（只写，参考下面的链下索引）访问。 此存储不填充在区块链网络中，也不需要对其进行共识计算。

由于在每个块导入期间都会产生一个脱链工作线程，因此在任何给定时间可能有多个脱链工作线程在运行。 因此，与任何多线程编程环境类似，还有一些实用程序可以在访问存储时对存储进行互斥锁以实现数据一致性。

链下存储充当各种链下工作线程相互通信以及在链下和链上逻辑之间进行通信的桥梁。 它也可以使用远程过程调用 (RPC) 读取，因此它适合存储无限增长的数据的用例，而不会过度消耗链上存储。

链下索引（Off-chain indexing）

区块链背景下的存储主要是关于链上状态。 但它很昂贵（因为它被填充到网络中的每个节点）并且不建议用于随着时间无限增长的历史或用户生成的数据。

为此，我们有链下存储。 除了 OCW 可以访问之外，Substrate 还包括一个称为“链下索引”的功能，允许运行时独立于 OCW 直接写入链下存储。 启动 Substrate 节点时，节点必须通过--enable-offchain-indexing标志选择加入此数据的持久性。

与在初始区块链同步期间不执行的 OCW 不同，链下索引每次处理一个块时都会填充存储，因此数据始终是一致的，并且对于启用索引的每个节点来说都是完全相同的。
