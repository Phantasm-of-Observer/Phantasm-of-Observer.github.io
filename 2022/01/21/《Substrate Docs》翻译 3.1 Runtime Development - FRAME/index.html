《Substrate Docs》翻译 3.1 Runtime Development - FRAME

框架（FRAME）

模块化实体运行时聚合框架 (FRAME) 是一组简化运行时开发的模块和支持库。 在Substrate中，这些模块称为Pallets，每个pallet都提供特定用处的逻辑并包含在Substrate链的运行时中。

FRAME还提供了一些帮助模块来与重要的[Substrate Primitives](sc_service - Rust (substrate.io))进行交互，这些Substrate Primitives提供了与核心客户端的接口。

托盘（Pallets）

使用FRAME构建时，Substrate 运行时由几个称为托盘的较小组件组成。托盘包含一组类型、存储项和功能，它们为运行时定义了一组特性和功能。

提示

如果您刚刚开始使用 Substrate 运行时开发，我们建议您尝试我们的介绍性教程来[创建您的第一个 Substrate 链](Create your first Substrate blockchain | Substrate_)。

FRAME 不仅提供了一个常用的 Substrate 托盘库，还提供了一个构建自定义域特定托盘的框架，使运行时工程师能够根据目标用例灵活地定义其运行时的行为。 结果是每个托盘都有自己的离散逻辑，可以修改区块链状态转换函数的特性和功能。

例如，[FRAME](Glossary | Substrate_)中包含的余额托盘（Balances pallet）定义了区块链的加密货币功能。 更具体地说，它定义：

  • 存储关于跟踪用户所拥有tokens的项。
  • 用户可以调用的函数来转移和管理这些代币。
  • 允许其他托盘使用这些tokens及其功能的API。
  • 允许其他托盘在用户余额发生变化时触发函数调用的钩子（Hook）。

Substrate 运行时工程师可以通过编写自己的托盘并通过将自定义托盘与现有的 FRAME 托盘或类似的 Substrate 模块组合来封装其区块链所需的功能，为其区块链定义自定义逻辑。 以下文档将向您展示如何操作。

托盘骨架（Skeleton of a pallet）

FRAME 托盘由 7 个部分组成：

  1. 导入和依赖
  2. 托盘类型声明
  3. 运行时配置Trait
  4. 运行时存储
  5. 运行时事件
  6. 钩子（Hooks）
  7. 外在（Extrinsics）

Pallets 可以根据需要由多个部分组成，从而在上述基本框架之上为运行时工程师提供了很大的灵活性。 请参阅[Substrate 运行时宏](Macros | Substrate_)以了解有关向 FRAME 托盘添加功能的更多信息。

系统板条箱（System crate）

[FRAME System crate frame_system](frame_system - Rust (substrate.io))为您的区块链提供低级类型、存储和功能。 所有其他托盘都依赖于 System crate 作为 Substrate 运行时的基础。

System crate 定义了 Substrate 运行时的所有核心类型，例如：

  • Origin
  • 区块号（Block Number）
  • 账户Id（Account Id）
  • 散列（Hash）
  • 区块头（Header）
  • 版本（Version）
  • 等等……

它还具有许多对系统至关重要的存储项，例如：

  • 账户随机数（Account Nonce）
  • 区块散列（Block Hash）
  • 区块号（Block Number）
  • 事件（Events）
  • 等等……

最后，它定义了许多低级函数，这些函数可以访问您的区块链存储、验证外部数据的来源等等。

支持板条箱（Support crate）

[FRAME Support Crate frame_support](frame_support - Rust (substrate.io)) 是 Rust 宏、类型、特征和模块的集合，可简化 Substrate 托盘的开发。

在编译时扩展的支持宏会生成托盘通用结构所需的样板代码，并使开发人员免于重复编写它们。

执行托盘（Executive pallet）

[FRAME 执行托盘 frame_executive](frame_executive - Rust (substrate.io))充当运行时的编排层。 它将传入的外部调用分派到运行时中的各个托盘。

运行时（Runtime）

运行时库汇集了所有这些组件和托盘。 它定义了哪些托盘包含在您的运行时中，并将它们配置为一起工作以构成您的最终运行时。 当对您的运行时进行调用时，它会使用 Executive 托盘将这些调用分派到各个托盘。

预制托盘（Prebuilt pallets）

一些托盘将具有足够的通用性，可以在许多区块链中重复使用。 任何人都可以自由编写和分享有用的托盘。 Substrate 提供了一系列常用功能的托盘。 让我们来探索一下。

资产（Asserts）

资产托盘是一个简单、安全的模块，用于处理可替代资产。

原子交换（Atomic Swap）

用于以原子方式将资金从来源发送到目标的模块。 证明用于允许目标批准（声明）交换。 如果在指定的时间内没有要求交换，发件人可以取消它。

Aura

Aura 托盘通过管理离线报告来扩展 Aura 共识。

权威发现（Authority Discovery）

权威发现托盘被核心/权威发现用来检索当前的权威集合，学习它自己的权威ID，以及签署和验证与其他权威之间的消息。

作者身份（Authorship）

Authorship 托盘跟踪区块的当前作者和最近的叔块（uncle）。

BABE

BABE 托盘通过从 VRF 输出收集链上随机性和管理 epoch 转换来扩展 BABE 共识。

余额（Balances）

余额托盘提供处理帐户和余额的功能。

基准（Benchmark）

以隔离方式包含常见运行时模式的托盘。 此托盘不用于生产区块链，仅用于基准测试和测试目的。

集体（Collective）

集体托盘允许一组帐户 ID 通过来自特定来源的可调度调用来表达他们的集体感受。

合同（Contracts）

Contracts 托盘为运行时提供了部署和执行 WebAssembly 智能合约的功能。

民主（Democracy）

民主托盘提供了一个民主系统，用于处理一般利益相关者投票的管理。

选举短语（Elections Phragmén）

Phragmén Elections 托盘是基于顺序 Phragmén的选举模块。

选举（Elections）

警告：未积极维护 选举托盘是一个选举模块，用于集体的权益加权成员选择。

示例链下工作机（Example Off chain Worker）

链下工作机示例：一个简单的托盘，展示了链下工作机共有的概念、API 和结构。

示例（Example）

示例托盘是一个简单的托盘示例，展示了大多数托盘共有的概念、API 和结构。

GRANDPA

GRANDPA 托盘通过管理为本地代码准备的 GRANDPA 权限集来扩展 GRANDPA 共识。

身份（Identity）

一个联合命名系统，允许从指定的来源添加多个注册商。 注册服务商可以设置费用来提供身份验证服务。 任何人都可以提出定期存款的提议身份，并要求任意数量的注册商进行审查（支付他们的每一项费用）。 注册服务商的判决以枚举的形式给出，允许复杂的、多层次的意见。

我在线（I‘m Online）

I’m Online 托盘允许验证者在每个新会话中八卦（gossip）传播一个心跳交易，以表明节点在线。

指数（Indices）

索引托盘为新创建的帐户分配索引。 索引是地址的缩写形式。

会员资格（Membership）

Membership 托盘允许控制一组 AccountIds 的成员身份，这对于管理集体的成员身份很有用。

多重签名（Multisig）

用于进行多签名调度的模块。

Nicks

Nicks 是一个简单的模块，用于跟踪链上的帐户名称。 它无需努力创建名称层次结构、替代 DNS 或提供反向查找。

违法行为（Offences）

违法行为托盘跟踪报告过的违法行为。

代理（Proxy）

一个模块，允许帐户授予其他帐户从其签名来源分派类型呼叫的权限。

随机性集体翻转（Randomness Collective Flip）

Randomness Collective Flip 托盘提供了一个可用于测试的随机函数，并根据前 81 个块的块哈希生成低影响随机值。 此托盘不适用于生产。

恢复（Recovery）

恢复托盘是一个 M-of-N 社交恢复工具，如果私钥或其他身份验证机制丢失，用户可以访问他们的帐户。 通过这个托盘，用户可以代表他们恢复的另一个帐户拨打电话。 恢复过程受到原始帐户所有者选择的受信任“朋友”的保护。 需要 N 个朋友中的一个阈值 (M) 才能让另一个帐户访问可恢复帐户。

调度器（Scheduler）

该模块公开了调度调度以在指定的块号或指定的时间段发生的能力。 这些预定的调度可以是命名的或匿名的，并且可以被取消。

计分池 （Scored Pool）

评分池托盘维护一个评分会员池，其中得分最高的实体成为会员。

会话（Session）

会话托盘允许验证者管理他们的会话密钥，提供更改会话长度的功能，并处理会话轮换。

社会（Society）

社会模块是一种经济游戏，它激励用户参与和维护会员社会。

质押（Staking）

质押托盘用于管理网络维护者的质押资金。

Sudo

Sudo 托盘允许单个帐户（称为“sudo 密钥”）执行需要Root来源或指定新帐户来替换它们作为 sudo 密钥的可调度功能。

时间戳（TimeStamp）

时间戳托盘提供获取和设置链上时间的功能。

交易支付（Transaction Payment）

交易支付托盘提供了计算发货前交易费用的基本逻辑。

财政（Treasury）

财政部托盘提供了一个可以由系统中的利益相关者管理的资金“罐”，以及用于从该罐中提出支出建议的结构。

公用事业（Utility）

带有用于调度管理的助手的无状态模块。

归属（Vesting）

一个简单的模块，提供一种在账户锁定余额上放置线性曲线的方法。 该模块可确保锁定到位，以防止余额因交易费支付以外的任何原因跌至未归属金额以下。
