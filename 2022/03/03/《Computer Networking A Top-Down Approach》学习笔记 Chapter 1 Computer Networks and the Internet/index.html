Chapter 1 Computer Networks and the Internet

What Is the Internet?

ISP：全称Internet Service Provider，即互联网服务提供商

regional：地方的，区域的

enterprise：企业

layer：层

thermostat：恒温器

A Nuts-and-Bolts Description

互联网是一个计算机网络，它连接了全世界数十亿的计算设备（The Internet is a computer network that interconnects billions of computing devices throughout the world）。

在之前，传统的计算设备仅仅指桌面电脑、Linux工作站等；而现在，笔记本电脑、智能手机、手表、家用电器等非传统的网络设备也被连接到互联网中。在网络术语中，这些统称为主机（hosts）或终端（end systems）。

终端系统通过通信链路（communication links）和分组交换机（packets switches）组成的网络连接在一起。

存在很多种通信链路，分别由不同的物理介质构成，包括电缆、铜线、光纤和无线电频谱等。不同的链路可以以不同的速率传输数据，链路的传输速率（transmission rate）以比特每秒来衡量。当一个终端系统有数据要发送到另一个终端系统时，发送端系统对数据进行分段，并在每个分段中添加标头字节。 生成的信息包，在计算机网络术语中称为数据包（packets），然后通过网络发送到目标端系统，在那里它们被重新组合成原始数据。

分组交换机接收到达其传入通信链路之一的数据包，并在其传出通信链路之一上转发该数据包。分组交换机有多种形式和风格，但当今 Internet 中最突出的两种类型是路由器（routers）和链路层交换机（link-layer switches）。两种类型的交换机都将数据包转发到它们的最终目的地。 链路层交换机通常用于接入网络，而路由器通常用于网络核心。 数据包从发送端系统到接收端系统所经过的通信链路和数据包交换机的序列称为通过网络的路由（route）或路径（path）。

分组交换网络（传输数据包）在许多方面类似于高速公路、道路和十字路口（传输车辆）的传输网络。 例如，一家工厂需要将大量货物运送到数千公里外的某个目的地仓库。 在工厂，货物被分割并装载到卡车车队中。 然后，每辆卡车独立地通过高速公路、道路和十字路口的网络到达目的地仓库。 在目的地仓库，货物被卸货并与来自同一批货物的其余货物组合在一起。 因此，在许多方面，数据包类似于卡车，通信链路类似于高速公路和道路，数据包交换机类似于十字路口，终端系统类似于建筑物。 就像卡车通过运输网络走一条路径一样，一个数据包通过计算机网络走一条路径。

终端系统通过ISP访问互联网。每个 ISP 本身就是一个由数据包交换机和通信链路组成的网络。 ISP 为终端系统提供各种类型的网络接入。 ISP 还为内容提供商提供 Internet 访问，将网站和视频服务器直接连接到 Internet。互联网就是将终端系统相互连接起来，因此提供终端系统访问权限的 ISP 也必须相互连接。这些下层 ISP 通过国家和国际上层 ISP相互连接。上层 ISP 由与高速光纤链路互连的高速路由器组成。每个 ISP 网络，无论是上层还是下层，都是独立管理的，运行 IP 协议（见下文），并符合某些命名和地址约定。

终端系统、分组交换机和 Internet 的其他部分运行控制 Internet 内信息发送和接收的协议（protocols）。传输控制协议 (TCP) 和 Internet 协议 (IP) 是 Internet 中最重要的两个协议。IP协议指定在路由器和终端系统之间发送和接收的数据包的格式。Internet 的主要协议统称为 TCP/IP。

鉴于协议对 Internet 的重要性，每个人都同意每个协议的作用非常重要，这样人们才能创建互操作的系统和产品。 这就是标准发挥作用的地方。 互联网标准（Internet standards）由互联网工程任务组 (IETF) 开发。 IETF 标准文档称为**征求意见稿 （requests for comments，简称RFCs)**。 RFCs 最初是作为征求意见的一般请求（因此得名）来解决互联网前身面临的网络和协议设计问题。RFC 往往非常技术性和详细。 它们定义了诸如 TCP、IP、HTTP（用于 Web）和 SMTP（用于电子邮件）等协议。

A Services Description

除了电子邮件和网上冲浪等传统应用程序外，互联网应用程序还包括移动智能手机和平板电脑应用程序等。这些应用程序被称为分布式应用程序（distributed applications），因为它们涉及多个相互交换数据的终端系统。Internet 应用程序在终端系统上运行——它们不在网络核心的分组交换机中运行。 尽管分组交换机促进了终端系统之间的数据交换，但它们并不关心作为数据源或接收器的应用程序。

让我们进一步探讨一下我们所说的为应用程序提供服务的基础设施是什么意思。 为此，假设您对分布式 Internet 应用程序有一个令人兴奋的新想法，该想法可能会极大地造福人类，或者可能只会让您变得富有和成名。 因为应用程序在终端系统上运行，您将需要编写在终端系统上运行的程序。 例如，您可以用 Java、C 或 Python 编写程序。 现在，因为您正在开发一个分布式 Internet 应用程序，运行在不同终端系统上的程序将需要相互发送数据。 在这里，我们遇到了一个中心问题——在一个终端系统上运行的一个程序如何指示 Internet 将数据传递给在另一个终端系统上运行的另一个程序。

连接到 Internet 的终端系统提供了一个套接字接口（socket interface），该接口指定在一个终端系统上运行的程序如何要求 Internet 基础设施将数据传递到在另一个终端系统上运行的特定目标程序。 这个 Internet 套接字接口是发送程序必须遵循的一组规则，以便 Internet 可以将数据传递给目标程序。假设 Alice 想使用邮政服务向 Bob 发送一封信。 当然，爱丽丝不能只写信（数据）然后把信扔出窗外。 相反，邮政服务要求爱丽丝把信放在信封里。 在信封中央写下 Bob 的全名、地址和邮政编码； 密封信封； 在信封的右上角盖上邮票； 最后，将信封放入官方邮政信箱。 因此，邮政服务有其自己的“邮政服务接口”或一组规则，Alice 必须遵循这些规则才能让邮政服务将她的信件投递给 Bob。 类似地，Internet 有一个套接字接口，发送数据的程序必须遵循该接口才能让 Internet 将数据传递给将要接收数据的程序。

当然，邮政服务为其客户提供不止一种服务。 提供快递、收货确认、普通使用等 多项服务。 以类似的方式，互联网为其应用程序提供多种服务。 当您开发 Internet 应用程序时，您也必须为您的应用程序选择一种 Internet 服务。

What is a Protocol

A Human Analogy

当您想向某人约时间时，人类的协议（或至少是良好的举止）规定一个人首先要打招呼来开始与其他人的交流。 对“Hi”的典型响应是返回的“Hi”消息。 然后人们会以亲切的“嗨”响应作为指示，表明可以继续并约时间。 对最初的“嗨”的不同回应（例如“别打扰我！”或“我不会说英语”或一些无法打印的回复）可能表示不愿意或无法沟通。 在这种情况下，人类的协议是不约一天中的时间。 有时人们对一个问题根本没有反应，在这种情况下，人们通常会放弃约那个人的时间。请注意，在我们的人类协议中，我们发送了特定的消息，以及我们为响应收到的回复消息或其他事件（例如在给定的时间内没有回复）而采取的特定行动。 显然，发送和接收的消息，以及在发送或接收这些消息或发生其他事件时采取的行动，在人类协议中起着核心作用。如果人们运行不同的协议（例如，如果一个人有礼貌而另一个人没有，或者如果一个人理解时间的概念而另一个人不知道），那么这些协议就无法互操作，并且无法完成任何有用的工作。 在网络中也是如此——它需要两个（或更多）通信实体运行相同的协议才能完成任务。

让我们考虑第二个人类类比。 假设你在上课。 老师喋喋不休地谈论协议，你很困惑。 老师停下来问：“有什么问题吗？” （发送给所有不睡觉的学生并由其接收的消息）。 你举手（向老师传达一个隐含的信息）。 你的老师微笑着说：“好的。” （鼓励您提出问题的传输信息），然后您提出问题（将您的信息传输给您的老师）。您的老师会听到您的问题（接收您的问题信息）并给出答案（将回复发送给您）。再一次，我们看到消息的发送和接收，以及在发送和接收这些消息时采取的一组常规操作，是这个问答协议的核心。

Network Protocols

网络协议类似于人类协议，只是交换消息和采取行动的实体是某些设备的硬件或软件组件。Internet 中涉及两个或多个通信远程实体的所有活动都受协议控制。 例如，两台物理连接的计算机中的硬件实现协议控制两个网络接口卡之间“线路”上的位流；终端系统中的拥塞控制协议控制发送方和接收方之间的数据包传输速率； 路由器中的协议确定数据包从源到目的地的路径。

当您在 Web 浏览器中键入网页的 URL 时会发生什么。首先，您的计算机将向 Web 服务器发送连接请求消息并等待回复。Web 服务器最终会收到您的连接请求消息并返回连接回复消息（connection request message）。 知道现在可以请求 Web 文档后，您的计算机就会在 GET 消息中发送它想要从该 Web 服务器获取的 Web 页面的名称。 最后，Web 服务器将网页（文件）返回给您的计算机。

鉴于上面的人和网络示例，消息的交换以及在发送和接收这些消息时采取的操作是协议的关键定义元素：

协议定义了两个或多个之间交换的消息的格式和顺序通信实体，以及在传输或接收消息时采取的行动或其他事件。

（A protocol defines the format and the order of messages exchanged between two or more communicating entities, as well as the actions taken on the transmission and/or receipt of a message or other event.）

Internet 和一般的计算机网络广泛使用协议。 不同的协议用于完成不同的通信任务。一些协议简单明了，而另一些协议则复杂且深奥难懂。 掌握计算机网络领域就等于了解网络协议是什么、为什么和如何操作。

The Network Edge

在计算机网络术语中，连接到 Internet 的计算机和其他设备通常被称为终端系统。它们被称为终端系统，因为它们位于 Internet 的边缘。

终端系统也称为主机，因为它们托管（即运行）应用程序，例如 Web 浏览器程序、Web 服务器程序、电子邮件客户端程序或电子邮件服务器程序。

主机有时进一步分为两类：客户端（clients）和服务器（servers）。 一般而言，客户端往往是台式机和移动 PC、智能手机等，而服务器往往是更强大的机器，用于存储和分发网页、流式传输视频、中继电子邮件等。 今天，我们从中接收搜索结果、电子邮件、网页和视频的大多数服务器都位于大型数据中心（data centers）。

1.2.1 Access Networks

接入网络——将终端系统物理连接到从终端系统到任何其他远程终端系统的路径上的第一个路由器（也称为“边缘路由器”）的网络【the access network—the network that physically connects an end system to the first router (also known as the “edge router”) on a path from the end system to any other distant end system】。

Home Access: DSL, Cable, FTTH, Dial-Up, and Satellite

今天，两种最流行的宽带住宅接入类型是数字用户线 ( digital subscriber line，即DSL) 和电缆。 住宅通常从提供有线本地电话接入的同一家本地电话公司 (telco) 获得 DSL 互联网接入。因此，当使用 DSL 时，客户的电信公司也是其 ISP。 如图所示，每个客户的 DSL 调制解调器使用现有的电话线（双绞铜线）与位于电信公司本地中心局 (CO) 的数字用户线接入多路复用器 (DSLAM) 交换数据。 家庭的 DSL 调制解调器接收数字数据并将其转换为高频音调，以便通过电话线传输到 CO； 来自许多此类住宅的模拟信号在 DSLAM 被转换回数字格式。
